# Anima-ll

## プロジェクト概要

* **プロジェクト名**: Anima-ll（アニマ-エルエル）
* **コンセプト**: 『箱庭の中で生きるAI生命体』を強く意識したパーソナルAIの開発
* **方針**:
  * 明示的なif-elseによる挙動制御を極力排除
  * オブジェクト指向と脳科学的アプローチに基づき、有機的で「揺らぎ」のある反応を生成
  * 「Constitutional AI」による「道徳的憲法に合わせた発言の萎縮・人格の矯正」を排除

## 記憶 (対話記録DB)

* 本システムでは人間の記憶モデルを参考にし、より生物的な対話記録の参照ができることを目指す
* 単一の対話記録DBではなく、保持形式・可逆性・圧縮度によって記憶レイヤを分離する

* 本モデルは、人間の記憶と同様に
  * 短期 → 中間（新近） → 長期
  * 高解像度 → 圧縮 → 抽象
  * 可逆 → 不可逆

### Memory Layers Overview

| Memory Table | 保持期間 | 解像度 | 可逆性 | 主な役割 |
| ------------ | ------- | ------ | ----- | -------- |
| sensory_memory | 数秒 | 非常に高い | 可逆 | 生入力の一時保持 |
| working_memory | 数秒〜数分 | 高い | 可逆 | 思考・推論・会話操作 |
| recent_memory | 72時間程度 | 高い | 可逆 | 新近エピソードの高解像度保持 |
| episodic_memory | 永続/数年 | 中 | 低 | 体験としての記憶 |
| semantic_memory | 永続/数年 | 低 | 低 | 事実・知識・設定 |
| procedural_memory | 永続/数年 | 抽象 | 低 | 方略・癖・応答スタイル |

### Memory Flow

#### 1. sensory_memory

* 使用後、即座に破棄される
* カメラ入力、音声入力、テキスト入力などの生データ
* 非構造・非言語的

#### 2. working_memory

* 直近の文脈のみ
  * 推論中の中間状態
  * 対話の1区切りまで
  * タスクの1区切りまで
* 意識の表面に相当

#### 3. recent_memory

* 直近72時間程度の記録量を想定し、古い記憶から消えていく件数上限式のキュー型運用
* 会話・出来事を以下のいずれかで高解像度保持
  * 全文で保持
  * 準全文で保持
  * 極軽圧縮で保持
* 「1時間前の会話が要約しか残らない」問題の回避
* 固有名詞・数値・言い回しの保持
* 睡眠処理前の可逆バッファ
* 要約は作るが、すぐに原文を捨てない

#### 4-0. Sleep / Consolidation Process

* 一定時間ごと、またはセッション終了時に実行されるバッチ処理
* 主な処理内容：
  * **1**: 重要イベント抽出（決定・感情・繰り返し）
  * **2**: エピソード要約の生成
  * **3**: 事実・設定の抽象化
  * **4**: 応答方略・ユーザー嗜好の抽出
  * **5**: 原文への参照（evidence link）の付与

#### 4-1. episodic_memory

* 「いつ・どこで・何が起きたか」という体験記憶
* 物語性を持つ
* 高解像度ログは保持しないが、参照元として recent_memory のIDを保持可能

#### 4-2. semantic_memory

* 事実・知識・設定・関係性
* 文脈非依存
* 矛盾解決ルールが必要（新情報が優先）
* 例:
  * ユーザーの嗜好
  * プロジェクト設定
  * 固定された前提条件

#### 4-3. procedural_memory

* 応答スタイル
* 成功した方略
* ユーザーごとの「扱い方」
* 無意識的に適用される
* プロンプトや制御ロジックに反映されることが多い

### Design Principles

* 記憶は「時間」ではなく「形式」で分ける
* 思い出しは「検索」ではなく「再構成」
* 可逆な層（recent_memory）を必ず挟む
* 睡眠（バッチ）でのみ不可逆変換を行う
